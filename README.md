**Описание проекта (вариант 1)**

Система для поиска оптимальных маршрутов между городами по трем критериям: длине, времени и стоимости. Алгоритм находит не один, а три оптимальных маршрута (по каждому критерию отдельно), а затем выбирает компромиссный маршрут на основе заданных приоритетов.

**Особенности реализации**

- Алгоритм Дейкстры с собственной реализацией минимальной кучи (MinHeap)
- Сложность O(E log V) для каждого критерия (длина, время, стоимость)
- Три независимых графа для оптимизации по разным критериям

**Производительность**

- Собственная реализация MinHeap без сторонних библиотек
- Эффективная работа с большими объемами данных
- Минимальные накладные расходы памяти
- Ранняя остановка алгоритма при достижении цели

**Структура проекта**

- exam.py # Основной файл с реализацией алгоритма
- input.txt # Входной файл с данными (города, дороги, запросы)
- output.txt # Выходной файл с результатами

**Требования к запуску**

Python 3.6 или выше
Отсутствие сторонних зависимостей

**Запуск программы**

Создайте файл input.txt с данными

Запустите скрипт:
`bash
exam.py
Результаты появятся в файле output.txt`

**Ограничения**

- Алгоритм не поддерживает отрицательные веса ребер
- Все дороги считаются двусторонними
- Входные данные должны быть корректно отформатированы
